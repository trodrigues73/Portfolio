Développement Python & Intégration par API

Dans le cadre de mon poste au Ministère des Armées, j'ai conçu et développé en Python un outil interne destiné à automatiser la création, la gestion et le suivi de règles de détection de comportements malveillants.

Ce projet reposait principalement sur l'interconnexion d'outils internes via des API REST, dans un environnement sécurisé et restreint. L'objectif était d'automatiser l'ensemble du cycle de vie d'une règle, depuis sa création jusqu'à sa documentation et son suivi opérationnel.

Le développement incluait :

  • L'écriture de scripts Python pour piloter les interactions entre services,
  • La gestion des appels API pour automatiser les opérations entre les outils suivants :
    • Bitbucket : pour le stockage des règles et le suivi des versions (historique Git),
    • Splunk : pour pousser automatiquement les règles dans l'outil de détection,
    • Mattermost : pour notifier en temps réel les parties prenantes lors de la création ou modification d'une règle,
    • Confluence : pour générer automatiquement une page de documentation conforme à un format standardisé, à chaque nouvelle règle.

Cette expérience m'a permis d'acquérir une solide expertise en développement Python orienté automatisation, en manipulation d'API REST, en authentification sécurisée (gestion de tokens et headers), ainsi qu'en structuration de code modulaire et maintenable.

Elle m'a également sensibilisé à la rigueur nécessaire dans les environnements sensibles, où la traçabilité, la cohérence documentaire et la communication entre outils sont essentielles.
